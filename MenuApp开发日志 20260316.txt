事件驱动的响应式编程模型（类似 React/Vue）

1. init.py 的核心作用
__init__.py 文件在 Python 中有以下重要作用：
✅ 将目录标识为 Python 包
让Python知道这个目录是一个可导入的包（package）
没有它，无法使用 from mobile import xxx 这样的导入语句
✅ 控制包的导出内容
可以在 __init__.py 中定义 __all__ 来指定哪些模块可以被导出
✅ 初始化包级别的变量
可以设置包级别的全局变量或配置
2. 空的 init.py 是完全正常的
在 Python 3.3+ 中，空的 __init__.py 文件是完全正常且常见的。它们的作用是：
仅仅作为包的标记
不需要任何代码就能工作
